# AI 求职助手 - 智能求职管理平台

> 基于 AI 的一站式求职管理平台，覆盖简历管理、岗位匹配、简历优化、投递追踪、模拟面试全流程。

## 项目概览

AI 求职助手是一款面向互联网行业实习/校招求职者的 Web 应用，帮助用户高效管理求职全流程。平台通过 Claude AI 实现简历智能解析、针对性优化建议、模拟面试对话与评估等核心能力，同时提供岗位匹配算法和投递进度管理等工具型功能。

### 核心价值

```
岗位匹配 → 简历优化 → 投递管理 → 面试准备 → 面试复盘
```

### 目标用户

- 在校生寻找暑期/日常实习
- 应届毕业生参加校招（秋招/春招）
- 转行进入互联网行业的求职者

## 功能模块详细说明

### 模块一：简历管理

**上传与解析**
- 支持 PDF 和 DOCX 格式上传（10MB 限制）
- 拖拽上传 + 进度条实时显示
- Claude AI 自动解析为结构化数据：
  - 基本信息（姓名、电话、邮箱）
  - 教育背景（学校、学位、专业、GPA、时间）
  - 经历（实习/项目/科研/竞赛，含职责描述、技能、量化指标）
  - 技能清单、获奖情况、证书

**多版本管理**
- 基于已有版本创建新版本（如"产品岗版本"、"技术岗版本"）
- 版本间 diff 对比视图（逐字段高亮差异）
- 重命名、复制、删除操作
- 解析后自动创建"默认版本"

**PDF 导出**
- 专业简洁的 A4 排版模板
- 支持中文字体（Noto Sans SC）
- 版本选择器 + 在线预览 + 一键下载

---

### 模块二：岗位匹配

**岗位数据**
- 预置 20 家头部互联网公司、40+ 岗位数据
- 支持手动添加岗位（完整 JD 录入表单）
- 搜索（公司名/岗位名）+ 筛选（城市/类型）+ 分页

**智能匹配算法**

基于五维度加权的关键词匹配引擎：

| 维度 | 权重 | 匹配逻辑 |
|------|------|----------|
| 技能匹配 | 30% | 简历技能 vs JD 要求关键词重合度 |
| 经历相关性 | 25% | 经历描述 vs 岗位职责文本相似度 |
| 硬性条件 | 20% | 城市、求职类型、学历规则匹配 |
| 行业/方向 | 15% | 用户意向 vs 岗位方向吻合度 |
| 优先条件 | 10% | 加分项与用户背景匹配 |

- 匹配度百分比 badge（绿 ≥80% / 黄 50-79% / 红 <50%）
- 展开查看匹配详情：命中技能（绿色标签）、缺失技能（红色标签）
- 支持按匹配度/时间排序切换

---

### 模块三：AI 简历优化

用户选择目标岗位后，Claude AI 生成针对性优化方案：

**优化方案五部分结构：**

1. **整体策略** — 岗位核心考察能力 vs 简历当前突出能力，调整方向
2. **逐条修改建议** — 每条经历的保留/删除/重写建议，含原文、改写文、修改理由、关键词命中分析
3. **关键词补充建议** — JD 中出现但简历缺失的关键词，建议添加位置
4. **经历取舍建议** — 哪些经历与岗位强相关（突出）、哪些无关（弱化）
5. **排版顺序建议** — 按相关性重新排列经历顺序

**交互方式：**
- 每条修改建议可独立接受/拒绝（checkbox）
- 全选/全不选快捷操作
- 「应用选中建议」一键生成新版本简历（自动命名"针对XX公司-XX岗位优化版"）

### 模块四：投递管理

**基础表格（TanStack Table）**

| 字段 | 说明 |
|------|------|
| 公司/岗位/部门 | 基本信息 |
| 投递时间 | 自动记录 |
| 进度 | 已投递→简历筛选→笔试→一面→二面→三面→HR面→Offer/已拒 |
| 优先级 | P0 必冲 / P1 重点 / P2 保底 / P3 海投 |
| 城市/备注 | 辅助信息 |

**高级功能：**
- 内联编辑（点击单元格直接修改进度/优先级/备注）
- 状态/优先级筛选下拉
- 批量操作（选中多行 → 批量更新进度/优先级/删除）
- 数据看板：投递总数、各阶段数量、本周新增
- 转化漏斗图：投递 → 笔试 → 一面 → 二面 → 三面 → HR面 → Offer
- 导出 CSV（Excel 兼容，含中文列名）

**与岗位模块联动：**
- 岗位详情页「标记已投递」按钮自动创建投递记录

---

### 模块五：模拟面试

**面试设置：**
- 选择目标岗位（自动加载 JD 作为面试上下文）
- 选择面试轮次：
  - 第一轮（技术/业务基础面）：60% 简历深挖 + 40% 八股基础
  - 第二轮（深度思维面）：50% 决策逻辑追问 + 30% 开放性问题 + 20% 行业认知
  - 第三轮（HR 综合面）：稳定性 + 价值观 + 综合素养 + 压力测试
- 选择面试官风格：

| 风格 | 特点 |
|------|------|
| 温和引导型 | 给提示，追问温和，适合初次练习 |
| 注重细节型 | 对数据、时间线、技术细节追问深 |
| 严格压力型 | 频繁 challenge，语气直接 |
| 发散思维型 | 追问"还有呢"、"换个角度" |
| 沉默寡言型 | 反馈少，考验自驱表达 |

**面试体验：**
- 聊天式界面（面试官左侧、用户右侧）
- Claude AI 流式输出（SSE），逐字显示面试官提问
- 实时计时器（MM:SS 格式）
- 面试结束按钮

**AI 评估报告：**
- 逐题评分（1-10 分）+ 回答亮点 + 改进建议 + 参考答案
- 综合评级：通过 / 待提升 / 不通过
- 整体优势与不足分析
- 后续改进建议

## 数据模型

```
User (用户)
├── Resume[] (简历)
│   ├── parsedData (AI 解析的结构化数据)
│   └── ResumeVersion[] (多版本)
├── JobPreference (求职意向, 1:1)
├── Application[] (投递记录)
│   ├── → Job (关联岗位)
│   └── → ResumeVersion (使用的简历版本)
└── MockInterview[] (模拟面试)
    ├── messages (对话历史)
    └── evaluation (AI 评估报告)

Job (岗位, 全局共享)
├── company, positionName, department
├── jobType (实习/校招/社招)
├── location[], requirements[], preferred[]
├── jdContent, salaryRange
└── createdBy → User (创建者)
```

## 快速开始

### 环境要求

- Node.js ≥ 20.9
- Docker & Docker Compose
- npm

### 安装步骤

```bash
# 1. 克隆项目
cd /home/tianfang/ai-job-assistant

# 2. 安装依赖（自动生成 Prisma 客户端）
npm install

# 3. 启动数据库
docker compose up -d

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env，填写 ANTHROPIC_API_KEY（AI 功能必须）

# 5. 同步数据库 schema
npx prisma db push

# 6. 导入种子数据（40 条示例岗位）
npm run prisma:seed

# 7. 启动开发服务器
npm run dev
```

访问 http://localhost:3000

### 环境变量说明

| 变量 | 必须 | 说明 |
|------|------|------|
| `DATABASE_URL` | 是 | PostgreSQL 连接串 |
| `NEXTAUTH_SECRET` | 是 | JWT 加密密钥（生产环境务必修改） |
| `NEXTAUTH_URL` | 是 | 应用 URL |
| `ANTHROPIC_API_KEY` | AI 功能必须 | Claude API 密钥，[获取地址](https://console.anthropic.com/) |

本项目使用 Anthropic Claude API（`claude-sonnet-4-6`）实现四个 AI 功能：

| 功能 | 触发方式 | 预期耗时 |
|------|----------|----------|

| 简历优化 | 岗位详情页点击「针对此岗位优化简历」 | 20-45s |
| 模拟面试 | 面试聊天中每轮回答后 | 3-5s/轮 |
| 面试评估 | 结束面试后自动生成 | 15-30s |

## 安全设计

- **认证**：NextAuth.js v5，JWT 会话策略，密码 bcrypt 哈希
- **路由保护**：proxy.ts 拦截所有未认证请求
- **API 安全**：每个 API 路由独立验证 session + 资源所有权
- **文件上传**：文件名清理（防路径穿越）+ MIME 类型验证 + 大小限制
- **输入验证**：Zod schema 验证所有用户输入，枚举值严格校验
- **消息限长**：面试聊天消息限制 5000 字
- **数据库索引**：所有外键列建立索引，防止全表扫描

## 开发方法论

本项目使用 [Vibe Coding](https://github.com/EnzeD/vibe-coding) 方法论开发：

1. **规划先行** — PRD → 技术栈文档 → 分步实施计划
2. **Memory Bank** — 所有规划文档存储在 `memory-bank/` 目录
3. **增量迭代** — 15 步计划，每步有明确的验证测试
4. **AI 辅助开发** — Claude Code 执行实现，人工审阅验证
5. **代码审查** — 每阶段进行规格合规 + 代码质量双重审查
