# 面试实时教练系统设计文档

## 概述

将现有的单栏聊天式模拟面试升级为**双栏面试教练系统**。左栏保持面试对话，右栏实时显示教练反馈（面试官 OS、维度评分、改进建议、推荐话术、一键重练）。新增错题本模块和面试能力雷达图。

核心差异化：相比直接用 ChatGPT 做面试练习，本产品提供**边练边学**的实时辅导体验，让用户在每一轮回答后立刻看到面试官视角、量化评分和可模仿的优秀回答。

---

## 架构设计

### 双次 AI 调用模型

用户每轮回答后，后端并行发起两个独立的 AI 请求：

```
用户回答
  ├─→ 面试官 AI（流式 SSE）→ 追问显示在左栏
  └─→ 教练 AI（非流式 JSON）→ OS/评分/建议/话术显示在右栏
```

- **面试官 AI**：使用现有的面试 system prompt（轮次 + 风格 + JD + 简历），流式输出追问
- **教练 AI**：独立的 system prompt，接收面试官问题 + 用户回答 + 简历 + JD，返回结构化 JSON
- 两个角色互不感知，面试官不知道教练说了什么，保持追问的自然感

### 页面布局

```
┌───────────────────────────────────────────────────────────┐
│  面试信息栏：岗位 | 轮次 | 风格 | 计时器 | 结束面试按钮     │
├────────────────────────────┬──────────────────────────────┤
│         左栏：面试对话        │       右栏：实时教练          │
│                            │                              │
│  [面试官] 问题 1             │  ┌─ 教练卡片 #1 ───────────┐ │
│  [用户] 回答 1               │  │ 面试官 OS               │ │
│  [面试官] 追问 2（流式）      │  │ 维度评分 (5条柱状)       │ │
│  [用户] 回答 2               │  │ 改进建议 (2-3条)         │ │
│  ...                       │  │ 推荐话术 (折叠)           │ │
│                            │  │ [一键重练]               │ │
│  ┌─────────────────────┐   │  └──────────────────────────┘│
│  │ 输入回答... [发送]    │   │  ┌─ 教练卡片 #2（折叠）──┐   │
│  └─────────────────────┘   │  └──────────────────────────┘│
├────────────────────────────┴──────────────────────────────┤
│              面试结束 → 能力雷达 + 总结报告                  │
└───────────────────────────────────────────────────────────┘
```

- 桌面端：左右并排，各占 50%
- 移动端：左栏为主视图，右栏通过底部抽屉或 Tab 切换访问
- 右栏最新的教练卡片展开，历史卡片自动折叠

## 教练卡片详细设计

每轮用户回答后生成一张教练卡片，包含以下五个区块（从上到下）：

### 1. 面试官 OS

融合两个视角的一段话：
- 这道题在考察什么能力
- 面试官对用户回答的真实反应

示例：
> *"这题在考察你的技术选型能力。你提到了用 Redis 做缓存，但没有解释为什么不用本地缓存或 Memcached，面试官会觉得缺少对比思维。提到了 QPS 数据是加分项。"*

样式：带特殊背景色和引号图标，营造"偷听面试官内心想法"的感觉。

### 2. 维度评分

五个维度，每个 1-10 分：

| 维度 | 说明 |
|------|------|
| 内容相关性 | 是否回答到了问题的核心考察点 |
| 结构清晰度 | 回答是否有框架（如 STAR），逻辑是否清晰 |
| 量化数据 | 是否有具体数字支撑论点 |
| 技术深度 | 技术细节是否到位（技术岗适用） |
| 表达流畅度 | 是否简洁连贯，没有冗余和跑题 |

- 综合分 = 五维度均分，显示在评分区域顶部
- 每个维度用水平条形图展示，颜色：绿(≥7) / 黄(5-6) / 红(≤4)
- 综合分 ≤ 4 时自动标记为错题，卡片上显示"已加入错题本"提示

### 3. 改进建议

2-3 条具体、可执行的 action item。

示例：
> *① 补充选型对比：说明 Redis vs Memcached vs 本地缓存的 trade-off*
> *② 加入量化：缓存命中率从 X% 提升到 Y%，响应时间降低了多少*
> *③ 补充踩坑经历：遇到过什么缓存穿透/击穿问题，怎么解决的*

### 4. 推荐话术

- 默认折叠，显示"查看推荐回答"按钮
- 点击展开，显示 AI 生成的**完整优秀回答**
- 基于用户的简历和经历生成（不是通用模板），用户可以直接学习和模仿
- 展开后可再次点击折叠

### 5. 一键重练

- 按钮在卡片底部
- 点击后的行为：
  1. 左栏：删除用户对该问题的回答及之后所有对话消息
  2. 右栏：删除该教练卡片及之后所有卡片
  3. 面试官重复同一个问题，等待用户重新回答
  4. 数据库：更新 messages 数组，裁剪到该问题之前的状态
- 恢复到"面试官刚问完这个问题"的干净状态

## 教练 AI 接口设计

### API 端点

`POST /api/interviews/coach`

请求体：
```json
{
  "interviewId": "string",
  "question": "string",
  "answer": "string",
  "round": "FIRST | SECOND | THIRD",
  "resumeData": {},
  "jobJd": "string"
}
```

响应体：
```json
{
  "os": "string",
  "scores": {
    "relevance": 8,
    "structure": 5,
    "quantification": 3,
    "technicalDepth": 7,
    "fluency": 6
  },
  "overallScore": 5.8,
  "suggestions": ["string", "string"],
  "referenceAnswer": "string",
  "isWeakPoint": true
}
```

### 教练 System Prompt 要点

- 角色：资深面试辅导教练，看过上万场面试
- 输入：面试官的问题、用户的回答、用户简历、岗位 JD、面试轮次
- 输出：严格 JSON 格式
- 评分标准：每个维度明确的 1-10 分定义
- 面试官 OS：要融合"考察意图"和"对回答的反应"两个视角，一段话说清
- 推荐话术：基于用户简历中的真实经历生成，不能编造用户没有的经历
- 改进建议：具体、可执行，不说"回答可以更好"这种废话

## 一键重练数据流

```
用户点击"重练第 N 轮"
  │
  ├─ 前端：
  │   ├ 左栏：删除第 N 轮用户回答及之后所有消息
  │   ├ 右栏：删除第 N 轮及之后所有教练卡片
  │   └ 左栏：显示面试官的第 N 轮问题（已存在，不需要重新生成）
  │
  └─ 后端 API：PATCH /api/interviews/[id]/rewind
      ├ body: { "rewindToIndex": number }
      ├ 裁剪 messages 数组到 rewindToIndex
      └ 裁剪 coachCards 数组对应项
```

## 错题本

### 页面位置

侧边栏独立入口，与简历管理、模拟面试并列：`/mistakes`

### 数据来源

- 自动加入：教练评分综合分 ≤ 4
- 手动收藏：用户点击教练卡片上的收藏按钮（任意评分均可收藏）

### 数据模型

在 MockInterview 的 coachCards JSON 中标记 `isWeakPoint` 和 `isBookmarked` 字段。错题本页面通过查询所有面试记录中标记的卡片来聚合展示。

```
WeakPoint 数据结构（存储在 coachCards JSON 数组中每个元素）:
{
  "questionIndex": number,
  "question": "string",
  "userAnswer": "string",
  "os": "string",
  "scores": { ... },
  "overallScore": number,
  "suggestions": ["string"],
  "referenceAnswer": "string",
  "isWeakPoint": boolean,
  "isBookmarked": boolean
}
```

### 错题本页面功能

- 列表视图：面试官问题、用户回答摘要（截断）、综合评分 badge、来源（公司+岗位）、日期
- 筛选：按维度筛选（如只看"量化数据"低分题）、按来源面试筛选
- 点击进入详情：完整教练卡片（OS + 评分 + 建议 + 推荐话术）
- 详情页提供"重练此题"按钮 → 创建一个新的迷你面试会话，只包含这一道题
- 支持手动移出错题本（取消标记）

## 能力雷达

### 触发时机

面试结束后自动生成，展示在面试评估报告页顶部。

### 数据计算

- 五个维度的分数 = 本次面试所有轮次各维度评分的均值
- 如果某些轮次被重练过，使用最后一次回答的评分

### 展示

- 五边形雷达图，纯 SVG 实现（不引入图表库）
- 每个顶点标注维度名称和分数
- 填充区域颜色：绿色（透明度 0.3）

### 与现有评估报告的关系

能力雷达替代现有评估报告中的"综合评级"部分，放在报告页最顶部。现有的逐题评分、亮点、改进建议等内容保留在下方（数据来源从现有的评估 AI 切换为教练卡片聚合）。
